---
title: "nivo link"
description: "Read .nivo, create or update the app on Nivo, and save the appId back to the file."
---

## Usage

```bash
nivo link
```

Reads the `.nivo` file in the current directory, creates (or updates) the app on the Nivo platform, and writes the generated `appId` back to `.nivo`.

## Required `.nivo` structure

```json .nivo
{
  "name": "api",
  "projectId": "PROJECT_ID",
  "sourceType": "zip",
  "type": "worker",
  "buildSystem": "nivopack",
  "runtime": "node20",
  "startCmd": "npm start",
  "ramMb": 100
}
```

After running `nivo link`, the file is updated with `appId` and `linkedAt`:

```json .nivo
{
  "name": "api",
  "projectId": "PROJECT_ID",
  "sourceType": "zip",
  "type": "worker",
  "buildSystem": "nivopack",
  "runtime": "node20",
  "startCmd": "npm start",
  "ramMb": 100,
  "appId": "77faa1f9-316f-4dbb-8c1c-41c8eb871346",
  "linkedAt": "2026-01-15T12:00:00.000Z"
}
```

See the [`.nivo` config reference](/config) for all available fields.
