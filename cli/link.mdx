## nivo link

```bash
nivo link
```

O comando `link` inicia um wizard interativo que:

1. Pergunta o nome do app
2. Lista os projetos do workspace para seleção
3. Permite criar um app novo ou conectar a um app existente
4. Salva o `appId` e `projectId` no arquivo `.nivo`

---

## O que é gravado no .nivo

Após o `link`, o arquivo `.nivo` recebe os campos automáticos:

```json .nivo
{
  "name": "meu-app",
  "projectId": "proj_abc123",
  "appId": "app_xyz789",
  "linkedAt": "2024-01-15T10:30:00.000Z",
  "startCmd": "npm start"
}
```

<Note>
  Os campos `appId` e `linkedAt` são preenchidos automaticamente pelo CLI. Não os edite manualmente.
</Note>

---

## Quando usar

<AccordionGroup>
  <Accordion title="Primeira vez no projeto">
    Execute `nivo link` antes do primeiro `nivo deploy`. O link cria o app na plataforma e configura o `.nivo`.
  </Accordion>
  <Accordion title="Conectar a um app existente">
    Se o app já existe no dashboard, o wizard permite selecionar um app existente em vez de criar um novo.
  </Accordion>
  <Accordion title="Relinkar após clonar o repositório">
    Se você clonou um repositório que tem `.nivo` mas sem `appId`, execute `nivo link` para reconectar.
  </Accordion>
</AccordionGroup>

---

<Tip>
  Você pode commitar o `.nivo` no repositório. O `appId` é um identificador público — não é um segredo.
</Tip>
