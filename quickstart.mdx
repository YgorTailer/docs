---

<Note>
  Você vai precisar de **Node.js 18+** e uma conta no [nivo.lat](https://nivo.lat) com um projeto criado.
</Note>

## 1. Instale o CLI

```bash
npm install -g @nivo-lat/cli
```

Confirme a instalação:

```bash
nivo --version
```

## 2. Faça login

```bash
nivo login
```

O CLI abre o navegador e exibe um código de verificação. Confirme o código para autenticar. Sua API key fica salva localmente em `~/.config/nivo/config.json`.

<Warning>
  Nunca compartilhe o arquivo `~/.config/nivo/config.json` — ele contém sua API key.
</Warning>

## 3. Configure o projeto

Crie o arquivo `.nivo` na raiz do seu projeto. Escolha o runtime conforme sua stack:

<Tabs>
  <Tab title="Node.js">
    ```json .nivo
    {
      "name": "meu-app-node",
      "projectId": "SEU_PROJECT_ID",
      "runtime": "node20",
      "installCmd": "npm install",
      "buildCmd": "npm run build",
      "startCmd": "npm start",
      "ramMb": 512
    }
    ```
  </Tab>
  <Tab title="Python">
    ```json .nivo
    {
      "name": "meu-app-python",
      "projectId": "SEU_PROJECT_ID",
      "runtime": "python3.11",
      "installCmd": "pip install -r requirements.txt",
      "startCmd": "python main.py",
      "ramMb": 512
    }
    ```
  </Tab>
  <Tab title="Bun">
    ```json .nivo
    {
      "name": "meu-app-bun",
      "projectId": "SEU_PROJECT_ID",
      "runtime": "bun1",
      "installCmd": "bun install",
      "buildCmd": "bun run build",
      "startCmd": "bun start",
      "ramMb": 256
    }
    ```
  </Tab>
  <Tab title="Dockerfile">
    ```json .nivo
    {
      "name": "meu-app-docker",
      "projectId": "SEU_PROJECT_ID",
      "buildSystem": "dockerfile",
      "dockerfilePath": "Dockerfile",
      "ramMb": 512
    }
    ```
  </Tab>
  <Tab title="Imagem Docker">
    ```json .nivo
    {
      "name": "meu-app-imagem",
      "projectId": "SEU_PROJECT_ID",
      "sourceType": "image",
      "buildSystem": "dockerimage",
      "dockerImage": "ghcr.io/minha-org/meu-app:latest",
      "ramMb": 512
    }
    ```
  </Tab>
</Tabs>

<Tip>
  Encontre o `projectId` no dashboard: abra seu projeto e copie o ID da URL ou nas configurações gerais.
</Tip>

## 4. Linke o projeto

```bash
nivo link
```

O CLI cria o app na plataforma e preenche o `appId` no `.nivo` automaticamente. Você só precisa fazer isso uma vez.

## 5. Faça o deploy

```bash
nivo deploy . --watch
```

Com `--watch`, os logs aparecem em tempo real até o deploy finalizar com sucesso ou erro.

<Tip>
  Omita `--watch` para um deploy silencioso que retorna imediatamente.
</Tip>

## O que acontece por baixo

<Steps>
  <Step title="Compactação">
    O CLI compacta o diretório atual em um `.zip`, ignorando `node_modules`, `.git` e outros arquivos desnecessários.
  </Step>
  <Step title="Upload">
    O `.zip` é enviado para os servidores do Nivo via upload multipart.
  </Step>
  <Step title="Build">
    O servidor executa `installCmd` e depois `buildCmd` dentro do container de build.
  </Step>
  <Step title="Inicialização">
    O app sobe com `startCmd`. O health check valida que a porta está respondendo.
  </Step>
  <Step title="URL pública">
    Uma URL `*.nivo.lat` é gerada e o tráfego é roteado automaticamente.
  </Step>
</Steps>

## Variáveis de ambiente

Configure variáveis antes do deploy pelo dashboard ou pelo SDK:

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  projectId: process.env.NIVO_PROJECT_ID!
})

await nivo.apps.updateEnv('SEU_APP_ID', {
  DATABASE_URL: 'postgres://...',
  SECRET_KEY: 'super-secreto'
})
```

<Warning>
  Após alterar variáveis de ambiente, faça um novo deploy para que as mudanças entrem em vigor.
</Warning>
