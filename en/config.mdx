---
title: ".nivo file"
description: "Every configuration field of a Nivo project's .nivo file."
---

The `.nivo` file lives at the root of your project and tells the CLI how the app should be created, built and deployed on the platform.

<Note>
  You can commit `.nivo` to your repository. It contains no secrets — only app metadata.
</Note>

---

## Fields

### Identification

<ParamField body="name" type="string" required>
  App name shown in the dashboard. Used to generate the public URL `name.nivo.lat`.
</ParamField>

<ParamField body="projectId" type="string" required>
  ID of the project the app belongs to. Found in the dashboard.
</ParamField>

<ParamField body="appId" type="string">
  App ID generated by `nivo link`. Filled in automatically — do not edit.
</ParamField>

---

### Build

<Note>
  With the default build (`nivopack`), the stack is **detected automatically** from your files (`package.json`, `requirements.txt`, `go.mod`, etc.). You don't need to declare the language or version.
</Note>

<ParamField body="buildSystem" type="string" default="nivopack">
  Build system to use.

  | Value | Description |
  |-------|-------------|
  | `nivopack` | Automatic build with stack detection (default). |
  | `dockerfile` | Uses the `Dockerfile` in the repository. |
  | `dockerimage` | Uses a public Docker image. |
</ParamField>

<ParamField body="installCmd" type="string">
  Command to install dependencies. E.g. `npm install`, `pip install -r requirements.txt`.
</ParamField>

<ParamField body="buildCmd" type="string">
  Command to compile the project. E.g. `npm run build`, `go build ./...`.
</ParamField>

<ParamField body="startCmd" type="string">
  Command to start the app in production. E.g. `npm start`, `python main.py`.
</ParamField>

---

### Docker

<ParamField body="dockerfilePath" type="string" default="Dockerfile">
  Path to the Dockerfile relative to the project root. Used when `buildSystem: "dockerfile"`.
</ParamField>

<ParamField body="dockerImage" type="string">
  Public Docker image URL. Used when `buildSystem: "dockerimage"`.

  ```json
  "dockerImage": "ghcr.io/my-org/my-app:latest"
  ```
</ParamField>

---

### Source code

<ParamField body="sourceType" type="string" default="zip">
  How the code reaches the server.

  | Value | Description |
  |-------|-------------|
  | `zip` | Upload via CLI (default). |
  | `github` | Clone from a GitHub repository. |
  | `image` | Public Docker image. |
</ParamField>

<ParamField body="repoFullName" type="string">
  GitHub repository in `org/repo` format. Used when `sourceType: "github"`.
</ParamField>

<ParamField body="repoBranch" type="string" default="main">
  Branch to use for the clone. Used when `sourceType: "github"`.
</ParamField>

---

### Network & Compute

<ParamField body="port" type="number" default="3000">
  Port the app listens on. Nivo forwards HTTP traffic to this port.
</ParamField>

<ParamField body="ramMb" type="number" default="512">
  Allocated RAM in MB. Common values: `256`, `512`, `1024`, `2048`.
</ParamField>
