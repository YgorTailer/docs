## How it works

<Steps>
  <Step title="Create a project in the dashboard">
    Go to [nivo.lat](https://nivo.lat), create your workspace and open a project. Copy the `projectId` from the URL or the project settings.
  </Step>
  <Step title="Install the CLI or SDK">
    <CodeGroup>
      ```bash CLI
      npm install -g @nivo-lat/cli
      ```
      ```bash SDK
      npm install @nivo-lat/sdk
      ```
    </CodeGroup>
  </Step>
  <Step title="Authenticate">
    <CodeGroup>
      ```bash CLI
      nivo login
      # Opens the browser — confirm the code shown in the terminal
      ```
      ```typescript SDK
      import { NivoAPI } from '@nivo-lat/sdk'

      const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
        projectId: process.env.NIVO_PROJECT_ID!
      })
      ```
    </CodeGroup>
  </Step>
  <Step title="Deploy or create resources">
    <CodeGroup>
      ```bash CLI — App
      nivo link        # creates the app and saves the appId in .nivo
      nivo deploy . --watch
      ```
      ```typescript SDK — Database
      const db = await nivo.databases.create({
        name: 'production',
        engine: 'postgresql',
        password: 'a-strong-password'
      })
      console.log(db.connectionString)
      ```
      ```typescript SDK — Storage
      await nivo.storage.buckets.create({
        name: 'uploads',
        public: true
      })
      const file = await nivo.storage.from('uploads').upload('logo.png', './logo.png')
      console.log(file.url)
      ```
    </CodeGroup>
  </Step>
</Steps>

## API errors

<AccordionGroup>
  <Accordion title="2xx — Success">
    | Status | Meaning |
    |--------|---------|
    | `200` | OK — request completed |
    | `201` | Created — resource created successfully |
    | `204` | No content — operation with no response body |
  </Accordion>
  <Accordion title="4xx — Client errors">
    | Status | Meaning |
    |--------|---------|
    | `400` | Bad request — invalid parameters |
    | `401` | Unauthorized — missing or invalid API key |
    | `403` | Forbidden — your plan does not allow this action |
    | `404` | Not found — resource does not exist |
    | `409` | Conflict — resource already exists |
    | `429` | Rate limit reached — wait for `Retry-After` |
  </Accordion>
  <Accordion title="5xx — Server errors">
    | Status | Meaning |
    |--------|---------|
    | `500` | Internal server error |
    | `502` | Bad gateway — node temporarily unavailable |
    | `503` | Service unavailable — try again |
  </Accordion>
</AccordionGroup>

## Rate limits

| Type | Limit |
|------|-------|
| Reads `GET` | 1,000 req/min |
| Writes `POST / PATCH / DELETE` | 300 req/min |
| File uploads | 100 MB/file |
| SDK `/api/sdk/v1` | 1,000 req/min per API key |

<Note>
  When the limit is reached, the API returns `429 Too Many Requests` with a `Retry-After` header indicating how many seconds to wait.
</Note>
