## How it works

Nivo uses a **prepaid balance + per-minute usage** model:

1. Your plan includes a **monthly balance** credited automatically every month
2. Each **active project** draws from your balance at its compute tier — **billed per minute**
3. If the balance runs out, the workspace is **paused** automatically (no surprise charges)
4. You can add **extra balance** at any time to reactivate

<Note>
  Compute billing is **per project**, per minute the project is active — not per app or database. Pause the project to stop consuming balance.
</Note>

All tiers are available on any plan. Billing is per minute of usage from your balance.

| Tier | vCPU | RAM | Price/hour | Best for |

| **ORBIT** | 1.5 | 2 GB | R$0.0138/h | Growing applications |

## Cost examples

Compute is billed **per project**, at the project tier, per minute the project is active — **not** per app or database. Multiple resources in the same project share the tier and the cost.

<AccordionGroup>
  <Accordion title="Project on the SPARK tier (0.75 vCPU / 768 MB)">
    - SPARK: R$0.0062/h
    - Project active 24h/day for 30 days: R$0.0062 × 720h = **R$4.46/mo**
    - Covered by the Pro balance (R$20): **R$15.54 left**
  </Accordion>
  <Accordion title="Project on the ORBIT tier (1.5 vCPU / 2 GB)">
    - ORBIT: R$0.0138/h
    - Project active 24h/day for 30 days: R$0.0138 × 720h = **R$9.94/mo**
    - With the Pro balance (R$20): **R$10.06 left**
  </Accordion>
  <Accordion title="Multiple resources in the same project">
    - Compute is billed **once per project**, at the project tier.
    - An ORBIT project with 3 apps + 1 database costs the same as an ORBIT project with 1 app: **R$0.0138/h**.

    <Tip>
      You pay for the project tier, not per individual resource. Apps and databases in the same project share the tier and compute cost.
    </Tip>
  </Accordion>
  <Accordion title="Occasional use">
    - Pause the project when you're not using it — compute is only billed while the project is active.
    - SPARK project active only 4h/day for 30 days: R$0.0062 × (4h × 30) = **R$0.74/mo**
  </Accordion>
</AccordionGroup>

<Note>
  **Storage** is billed separately, by the reserved capacity of the project's storage tier — independent of compute.
</Note>

## Spend Cap

The Pro plan includes Spend Cap enabled by default — you set a maximum monthly spend limit.

<Steps>
  <Step title="Set the limit">
    In **Workspace → Billing → Spend Cap**, define the maximum amount that can be spent beyond the included monthly balance.
  </Step>
  <Step title="Automatic alerts">
    You get notifications at 50%, 80% and 100% of the configured budget.
  </Step>
  <Step title="Workspace paused at the limit">
    When the Spend Cap is reached, the workspace pauses automatically — no extra charges.
  </Step>
</Steps>

## Automatic pauses

When the balance reaches **R$0.00**, the workspace is paused automatically.

- Apps and databases go offline
- Data is preserved
- Reactivate by adding balance in **Workspace → Billing → Add balance**
- Resources come back up automatically once the balance is credited

<Warning>
  Paused apps lose container state (in-memory variables, temporary files inside the container). Use Storage or databases for permanent persistence.
</Warning>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Does the monthly balance roll over?">
    No. The plan's monthly balance is credited on each renewal and does not roll over to the next month. Extra balance purchased manually **does not expire**.
  </Accordion>
  <Accordion title="Can I be charged beyond my balance?">
    No. The workspace is paused automatically when the balance reaches zero. There are no surprise charges — Nivo never bills beyond the available balance.
  </Accordion>
  <Accordion title="Does stopping a single app stop billing?">
    No. Compute is billed at the **project** tier while the project is active. Stopping a single app or database does not reduce compute cost — to stop billing, **pause the project**. Data is preserved while paused.
  </Accordion>
  <Accordion title="Do automatic backups consume balance?">
    No. Automatic backups and project snapshots on the Pro and Builder plans don't consume extra balance — they're included in the plan.
  </Accordion>
  <Accordion title="How do I change the compute tier?">
    In the dashboard, open the project and go to **Settings → Compute**. Select the new tier. The change applies to all project resources and is billed from the next minute.
  </Accordion>
  <Accordion title="Can the Free plan create projects?">
    No. The Free plan (`maxProjects: 0`) gives dashboard access but does not allow creating active projects. Subscribe to Pro or Builder to create projects with apps, databases and storage.
  </Accordion>
</AccordionGroup>
