## New features

**Private networking between services**

Apps and databases in the same project now communicate over a private network,
reaching each other **by name** — without exposing ports to the public internet.

- Databases now offer an **internal connection string**, ideal for use as your
  apps' `DATABASE_URL`.
- Each app gets an **internal address** (`internalUrl`) for direct
  service-to-service calls within the project.
- Workers stay fully private, reachable only inside the project.

**More complete Storage**

The storage module gained a fluent API and new capabilities:

- `from()` API to work with buckets more naturally.
- **Signed URLs** to grant temporary access to private files.
- Path-based uploads, plus **move, copy and batch delete**.
- Public buckets serve content directly, with optimized delivery.

**Secure password generator**

When creating a database or changing a password, you can now generate a strong
password with one click — using only characters that are safe for connection
strings, environment variables and the command line.
