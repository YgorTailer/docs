## Como funciona

A Nivo usa **saldo pré-pago + consumo por minuto**:

1. Seu plano inclui um **saldo mensal** creditado automaticamente todo mês
2. Cada **projeto ativo** consome do saldo pelo seu tier de compute — **cobrado por minuto**
3. Se o saldo acabar, o workspace é **pausado** automaticamente (sem cobrança surpresa)
4. Você pode adicionar **saldo adicional** a qualquer momento para reativar

<Note>
  A cobrança de compute é **por projeto**, por minuto em que o projeto está ativo — não por app ou banco. Pause o projeto para parar de consumir saldo.
</Note>

## Tiers de compute

Todos os tiers estão disponíveis em qualquer plano. A cobrança é por minuto de uso via saldo.

| Tier | vCPU | RAM | Preço/hora | Ideal para |
|------|------|-----|-----------|-----------|
| **SPARK** | 0,75 | 768 MB | R$0,0062/h | Testes e projetos hobby |
| **PULSE** | 1 | 1 GB | R$0,0076/h | Websites e APIs leves |
| **NOVA** | 1,25 | 1,5 GB | R$0,0111/h | Aplicações em crescimento |
| **ORBIT** | 1,5 | 2 GB | R$0,0138/h | Aplicações em crescimento |
| **NEBULA** | 2 | 4 GB | R$0,031/h | Bancos de dados de produção |
| **QUASAR** | 4 | 8 GB | R$0,062/h | Workloads de alta performance |
| **COSMOS** | 8 | 16 GB | R$0,125/h | Workloads máximos |

<Note>
  O tier de compute é por **projeto** — todos os apps e bancos de dados dentro do mesmo projeto compartilham o tier escolhido. Configure em **Projeto → Configurações → Compute**.
</Note>

## Exemplos de custo

O compute é cobrado **por projeto**, pelo tier do projeto, por minuto em que o projeto está ativo — **não** por app ou banco. Vários recursos no mesmo projeto compartilham o tier e o custo.

<AccordionGroup>
  <Accordion title="Projeto no tier SPARK (0,75 vCPU / 768 MB)">
    - SPARK: R$0,0062/h
    - Projeto ativo 24h/dia por 30 dias: R$0,0062 × 720h = **R$4,46/mês**
    - Coberto pelo saldo Pro (R$20): **sobram R$15,54**
  </Accordion>
  <Accordion title="Projeto no tier ORBIT (1,5 vCPU / 2 GB)">
    - ORBIT: R$0,0138/h
    - Projeto ativo 24h/dia por 30 dias: R$0,0138 × 720h = **R$9,94/mês**
    - Com saldo Pro (R$20): **sobram R$10,06**
  </Accordion>
  <Accordion title="Vários recursos no mesmo projeto">
    - O compute é cobrado **uma vez por projeto**, pelo tier do projeto.
    - Um projeto ORBIT com 3 apps + 1 banco custa o mesmo que um projeto ORBIT com 1 app: **R$0,0138/h**.

    <Tip>
      Você paga pelo tier do projeto, não por recurso individual. Apps e bancos do mesmo projeto compartilham o tier e o custo de compute.
    </Tip>
  </Accordion>
  <Accordion title="Uso esporádico">
    - Pause o projeto quando não estiver usando — o compute só é cobrado enquanto o projeto está ativo.
    - Projeto SPARK ativo apenas 4h/dia por 30 dias: R$0,0062 × (4h × 30) = **R$0,74/mês**
  </Accordion>
</AccordionGroup>

<Note>
  O **storage** é cobrado à parte, pela capacidade reservada do tier de storage do projeto — independente do compute.
</Note>

## Spend Cap

O plano Pro inclui Spend Cap ativado por padrão — você configura um limite máximo de gasto mensal.

<Steps>
  <Step title="Configure o limite">
    Em **Workspace → Cobrança → Spend Cap**, defina o valor máximo que pode ser gasto além do saldo mensal incluso.
  </Step>
  <Step title="Alertas automáticos">
    Você recebe notificações em 50%, 80% e 100% do orçamento configurado.
  </Step>
  <Step title="Workspace pausado ao atingir o limite">
    Quando o Spend Cap é atingido, o workspace pausa automaticamente — sem cobrança adicional.
  </Step>
</Steps>

## Pausas automáticas

Quando o saldo chega a **R$0,00**, o workspace é pausado automaticamente.

- Apps e bancos de dados ficam offline
- Dados são preservados
- Reative adicionando saldo em **Workspace → Cobrança → Adicionar saldo**
- Os recursos sobem automaticamente após o saldo ser creditado

<Warning>
  Apps em pausa perdem estado de container (variáveis em memória, arquivos temporários dentro do container). Use Storage ou bancos de dados para persistência permanente.
</Warning>

## Perguntas frequentes

<AccordionGroup>
  <Accordion title="O saldo mensal acumula?">
    Não. O saldo mensal do plano é creditado a cada renovação e não acumula para o próximo mês. O saldo adicional adquirido manualmente **não expira**.
  </Accordion>
  <Accordion title="Posso ser cobrado além do meu saldo?">
    Não. O workspace é pausado automaticamente quando o saldo chega a zero. Não há cobrança surpresa — a Nivo nunca cobra além do saldo disponível.
  </Accordion>
  <Accordion title="Parar um app individual interrompe a cobrança?">
    Não. O compute é cobrado pelo tier do **projeto** enquanto o projeto está ativo. Parar um app ou banco isolado não reduz o custo de compute — para interromper a cobrança, **pause o projeto**. Os dados são preservados durante a pausa.
  </Accordion>
  <Accordion title="Backups automáticos consomem saldo?">
    Não. Backups automáticos e snapshots do projeto nos planos Pro e Builder não consomem saldo adicional — estão inclusos no plano.
  </Accordion>
  <Accordion title="Como alterar o tier de compute?">
    No dashboard, abra o projeto e vá em **Configurações → Compute**. Selecione o novo tier. A mudança se aplica a todos os recursos do projeto e é cobrada a partir do próximo minuto.
  </Accordion>
  <Accordion title="O plano Free pode criar projetos?">
    Não. O plano Free (`maxProjects: 0`) dá acesso ao painel, mas não permite criar projetos ativos. Assine o Pro ou Builder para criar projetos com apps, bancos de dados e storage.
  </Accordion>
</AccordionGroup>
