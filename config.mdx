---
title: "Arquivo .nivo"
description: "Referência completa do arquivo de configuração .nivo usado pelo CLI."
---

O arquivo `.nivo` fica na raiz do seu projeto e define como o CLI deve criar e fazer deploy do app.

## Exemplo completo

```json .nivo
{
  "name": "meu-app",
  "projectId": "proj_abc123",
  "sourceType": "zip",
  "type": "site",
  "buildSystem": "nivopack",
  "runtime": "node20",
  "installCmd": "npm install",
  "buildCmd": "npm run build",
  "startCmd": "npm start",
  "ramMb": 256,
  "subdomain": "meu-app"
}
```

## Referência de campos

| Campo | Tipo | Obrigatório | Padrão | Descrição |
|-------|------|-------------|--------|-----------|
| `name` | string | Sim | — | Nome do app. |
| `projectId` | string | Sim | — | ID do projeto no dashboard. |
| `sourceType` | string | Não | `zip` | `zip` ou `github`. |
| `type` | string | Não | `site` | `site` ou `worker`. |
| `buildSystem` | string | Não | `nivopack` | `nivopack`, `nixpacks` ou `dockerfile`. |
| `runtime` | string | Não | `node20` | Runtime (ex: `node20`, `python3.11`). |
| `installCmd` | string | Não | — | Comando de instalação (ex: `npm install`). |
| `buildCmd` | string | Não | — | Comando de build (ex: `npm run build`). |
| `startCmd` | string | Não | `npm start` | Comando de inicialização. |
| `ramMb` | number | Não | `256` | RAM em MB. Mínimo: `100`. |
| `subdomain` | string | Não | — | Subdomínio personalizado (minúsculas, números, hífens). |
| `repoFullName` | string | Não | — | Repositório GitHub no formato `owner/repo`. |
| `repoBranch` | string | Não | `main` | Branch do Git para deploy. |
| `appId` | string | — | — | **Preenchido automaticamente** pelo `nivo link`. Não edite. |
| `linkedAt` | string | — | — | **Timestamp automático** do `nivo link`. Não edite. |

## Como usar

<Steps>
  <Step title="Crie o .nivo">
    Crie o arquivo na raiz do projeto com pelo menos `name` e `projectId`.
  </Step>
  <Step title="Execute nivo link">
    ```bash
    nivo link
    ```
    Cria o app e preenche o `appId` automaticamente.
  </Step>
  <Step title="Faça o deploy">
    ```bash
    nivo deploy .
    ```
  </Step>
</Steps>

<Note>
  Você pode commitar o `.nivo` no repositório. Nunca commite o arquivo local do CLI em `~/.config/nivo/config.json`, porque ele contém a API key da sua conta.
</Note>
