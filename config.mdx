---
title: "Arquivo .nivo"
description: "Todos os campos de configuração do arquivo .nivo de um projeto Nivo."
---

O arquivo `.nivo` fica na raiz do projeto e diz ao CLI como o app deve ser criado, buildado e implantado na plataforma.

<Note>
  Você pode commitar o `.nivo` no repositório. Ele não contém segredos — apenas metadados do app.
</Note>

---

## Campos

### Identificação

<ParamField body="name" type="string" required>
  Nome do app exibido no dashboard. Usado para gerar a URL pública `nome.nivo.lat`.
</ParamField>

<ParamField body="projectId" type="string" required>
  ID do projeto ao qual o app pertence. Obtido no dashboard.
</ParamField>

<ParamField body="appId" type="string">
  ID do app gerado pelo `nivo link`. Preenchido automaticamente — não edite.
</ParamField>

---

### Runtime & Build

<ParamField body="runtime" type="string">
  Runtime utilizado no build e execução. Ignorado se `buildSystem` for `dockerfile` ou `dockerimage`.

  | Valor | Versão |
  |-------|--------|
  | `node18` | Node.js 18 LTS |
  | `node20` | Node.js 20 LTS |
  | `node22` | Node.js 22 |
  | `bun1` | Bun 1.x |
  | `python3.11` | Python 3.11 |
  | `python3.12` | Python 3.12 |
  | `go1.22` | Go 1.22 |
  | `deno2` | Deno 2.x |
</ParamField>

<ParamField body="buildSystem" type="string" default="nivopack">
  Sistema de build a usar.

  | Valor | Descrição |
  |-------|-----------|
  | `nivopack` | Build automático por runtime (padrão). |
  | `dockerfile` | Usa o `Dockerfile` no repositório. |
  | `dockerimage` | Usa uma imagem Docker pública. |
</ParamField>

<ParamField body="installCmd" type="string">
  Comando para instalar dependências. Ex: `npm install`, `pip install -r requirements.txt`.
</ParamField>

<ParamField body="buildCmd" type="string">
  Comando para compilar o projeto. Ex: `npm run build`, `go build ./...`.
</ParamField>

<ParamField body="startCmd" type="string">
  Comando para iniciar o app em produção. Ex: `npm start`, `python main.py`.
</ParamField>

---

### Docker

<ParamField body="dockerfilePath" type="string" default="Dockerfile">
  Caminho para o Dockerfile relativo à raiz do projeto. Usado quando `buildSystem: "dockerfile"`.
</ParamField>

<ParamField body="dockerImage" type="string">
  URL da imagem Docker pública. Usado quando `buildSystem: "dockerimage"`.

  ```json
  "dockerImage": "ghcr.io/minha-org/meu-app:latest"
  ```
</ParamField>

---

### Fonte do código

<ParamField body="sourceType" type="string" default="zip">
  Como o código chega ao servidor.

  | Valor | Descrição |
  |-------|-----------|
  | `zip` | Upload via CLI (padrão). |
  | `github` | Clone do repositório GitHub. |
  | `image` | Imagem Docker pública. |
</ParamField>

<ParamField body="repoFullName" type="string">
  Repositório no GitHub no formato `org/repo`. Usado quando `sourceType: "github"`.
</ParamField>

<ParamField body="repoBranch" type="string" default="main">
  Branch a ser usado no clone. Usado quando `sourceType: "github"`.
</ParamField>

---

### Rede & Compute

<ParamField body="port" type="number" default="3000">
  Porta em que o app escuta. O Nivo encaminha o tráfego HTTP para esta porta.
</ParamField>

<ParamField body="ramMb" type="number" default="512">
  RAM alocada em MB. Valores comuns: `256`, `512`, `1024`, `2048`.
</ParamField>
