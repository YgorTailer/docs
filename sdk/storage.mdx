<Note>
  Use o storage sempre pelo backend com `NivoAPI`. Nunca exponha a API key no client-side — para o navegador, gere **signed URLs**.
</Note>

## Como funciona

Você cria um bucket, define se é público ou privado, e:

- **Bucket público** → arquivos servidos diretamente com URL permanente e cache imutável.
- **Bucket privado** → sem URL pública; o acesso é feito por **signed URL** temporária ou download pelo backend.

---

## Buckets

<Tabs>
  <Tab title="Público">
    Arquivos públicos são servidos com URL permanente e cache imutável.

    ```typescript
    const bucket = await nivo.storage.buckets.create({
      name: 'assets-publicos',
      public: true,
      cacheControl: 'public, max-age=31536000, immutable'
    })
    ```
  </Tab>
  <Tab title="Privado">
    Arquivos privados só são acessíveis por signed URL ou download autenticado.

    ```typescript
    const bucket = await nivo.storage.buckets.create({
      name: 'documentos-clientes',
      public: false,
      allowedMimeTypes: ['application/pdf', 'image/*'],
      fileSizeLimit: 50 * 1024 * 1024  // 50 MB
    })
    ```
  </Tab>
</Tabs>

### Gerenciar buckets

```typescript
const buckets = await nivo.storage.buckets.list()
const bucket  = await nivo.storage.buckets.get('bucket_id')

await nivo.storage.buckets.update('bucket_id', { fileSizeLimit: 10 * 1024 * 1024 })
await nivo.storage.buckets.delete('bucket_id')
```

## A API fluente `from()`

A forma recomendada de trabalhar com arquivos é pelo handle `from('nome-do-bucket')`. O bucket é resolvido pelo nome automaticamente e os arquivos são endereçados por **path**.

```typescript
const avatares = nivo.storage.from('avatares')

// Upload para um path exato
const file = await avatares.upload('users/123/foto.jpg', buffer)
console.log(file.url)   // URL pública se o bucket for público

// Listar
const files = await avatares.list({ search: 'foto', limit: 50 })

// Remover (em lote, por id)
await avatares.remove([file.id])
```

<Note>
  O upload por path **rejeita** se já existir um objeto naquele caminho. Para substituir, use `move` ou remova o antigo antes.
</Note>

## Upload

O primeiro argumento é sempre o **path** do objeto dentro do bucket.

<Tabs>
  <Tab title="Arquivo local">
    ```typescript
    const bucket = nivo.storage.from('avatares')
    const file = await bucket.upload('users/123/foto.jpg', './foto.jpg')
    console.log(file.url)
    ```
  </Tab>
  <Tab title="Buffer / Stream">
    ```typescript
    import { readFileSync } from 'fs'

    const buffer = readFileSync('./relatorio.pdf')
    const bucket = nivo.storage.from('relatorios')

    const file = await bucket.upload('2024/jan/relatorio.pdf', buffer, {
      mimeType: 'application/pdf'
    })
    ```
  </Tab>
  <Tab title="Express (req.file)">
    ```typescript
    import express from 'express'
    import multer from 'multer'

    const upload = multer({ storage: multer.memoryStorage() })
    const app = express()
    const bucket = nivo.storage.from('uploads')

    app.post('/upload', upload.single('file'), async (req, res) => {
      const file = await bucket.upload(
        `users/${req.user.id}/${req.file!.originalname}`,
        req.file!.buffer,
        { mimeType: req.file!.mimetype }
      )
      res.json({ id: file.id, url: file.url })
    })
    ```
  </Tab>
</Tabs>

<Tip>
  O path aceita pastas (`users/123/avatar.jpg`). Segmentos `..`, barras iniciais e caracteres inválidos são removidos automaticamente — não há risco de path traversal.
</Tip>

## Mover e copiar

Operações por path dentro do mesmo bucket.

```typescript
const bucket = nivo.storage.from('avatares')

// Mover/renomear (preserva o mesmo id do arquivo)
await bucket.move('temp/upload.jpg', 'users/123/avatar.jpg')

// Copiar (cria um novo arquivo)
await bucket.copy('templates/default.jpg', 'users/456/avatar.jpg')
```

<Note>
  `from` e `to` são paths dentro do bucket. Tanto o path puro (`users/1/a.jpg`) quanto a key completa (`avatares/users/1/a.jpg`) são aceitos. Mover/copiar para um path já existente retorna `409`.
</Note>

## Signed URL — acesso privado sem proxy

Para entregar um arquivo privado ao navegador, gere uma **signed URL** temporária no backend. O cliente baixa direto da Nivo, sem passar pelo seu servidor e sem expor a API key.

```typescript
app.get('/arquivos/:id', autenticar, async (req, res) => {
  // 1. Verifique permissão na sua aplicação
  const registro = await db.arquivos.findById(req.params.id)
  if (registro.userId !== req.user.id) {
    return res.status(403).json({ error: 'Sem permissão' })
  }

  // 2. Gere uma URL temporária
  const { signedUrl, expiresAt } = await nivo.storage.files.createSignedUrl(
    registro.nivoFileId,
    { expiresIn: 3600 }   // expira em 1 hora
  )

  // 3. Redirecione o usuário direto para o storage
  res.redirect(signedUrl)
})
```

<ParamField path="expiresIn" type="number" default="3600">
  Segundos até a URL expirar. Padrão 1 hora, máximo 7 dias.
</ParamField>

<ParamField path="inline" type="boolean" default="false">
  Quando `true`, o arquivo abre no navegador (preview) em vez de baixar.
</ParamField>

<Tip>
  Signed URLs funcionam tanto para buckets privados quanto públicos. Use-as sempre que precisar de um link temporário e controlado.
</Tip>

## Upload direto do cliente

Para deixar o navegador ou app enviar o arquivo **direto ao storage** sem passar pelo seu servidor, gere uma signed upload URL no backend.

<Steps>
  <Step title="Backend gera a URL">
    ```typescript
    const bucket = nivo.storage.from('uploads')

    const { uploadUrl, key, expiresAt } = await bucket.createSignedUploadUrl(
      'video.mp4',
      { path: 'users/123/video.mp4', maxBytes: 500 * 1024 * 1024, expiresIn: 600 }
    )

    res.json({ uploadUrl, key })
    ```
  </Step>
  <Step title="Cliente envia direto">
    ```typescript
    // No navegador / app
    const form = new FormData()
    form.append('file', fileBlob)

    await fetch(uploadUrl, { method: 'POST', body: form })
    ```
  </Step>
</Steps>

<ParamField path="path" type="string">
  Path exato do objeto no bucket. Omita para uma key gerada automaticamente.
</ParamField>

<ParamField path="maxBytes" type="number">
  Limite máximo de bytes aceito no upload.
</ParamField>

<ParamField path="expiresIn" type="number" default="600">
  Segundos até a URL de upload expirar. Padrão 10 minutos, máximo 1 hora.
</ParamField>

## Listar e remover

```typescript
const bucket = nivo.storage.from('avatares')

// Listar com filtros
const files = await bucket.list({ search: 'avatar', page: 1, limit: 50 })

// Remover um arquivo
await nivo.storage.files.delete('file_id')

// Remover vários de uma vez
const result = await bucket.remove(['file_1', 'file_2', 'file_3'])
console.log(`${result.deleted} de ${result.requested} removidos`)
```

## Download pelo backend

Quando você precisa dos **bytes** no servidor (processar, reenviar, etc.), use o download direto:

```typescript
const buffer = await nivo.storage.files.download('file_id')
```

<Note>
  Para entregar ao navegador, prefira `createSignedUrl` — evita que seu servidor vire gargalo em arquivos grandes.
</Note>

## Multipart upload

Para arquivos grandes (acima de 50 MB):

<Steps>
  <Step title="Iniciar">
    ```typescript
    const bucket = nivo.storage.from('backups')
    const mp = await bucket.startMultipart({
      filename: 'backup-2024.zip',
      mimeType: 'application/zip',
      totalBytes: fileSize
    })
    ```
  </Step>
  <Step title="Enviar partes">
    ```typescript
    const chunkSize = 10 * 1024 * 1024  // 10 MB
    const parts = []

    for (let i = 0; i * chunkSize < fileBuffer.length; i++) {
      const chunk = fileBuffer.slice(i * chunkSize, (i + 1) * chunkSize)
      const part = await bucket.uploadPart(mp.fileId, mp.uploadId, i + 1, chunk)
      parts.push(part)
    }
    ```
  </Step>
  <Step title="Finalizar">
    ```typescript
    const { file } = await bucket.completeMultipart(mp.fileId, mp.uploadId, parts)
    console.log(file.url)
    ```
  </Step>
</Steps>
