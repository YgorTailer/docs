## Criar banco de dados

O `create` retorna o banco já provisionado com a **connection string** completa no campo `connectionString`.

<Tabs>
  <Tab title="PostgreSQL">
    ```typescript
    const db = await nivo.databases.create({
      name: 'db-producao',
      engine: 'postgresql',
      password: 'uma-senha-forte'
    })

    console.log(`Host: ${db.host}`)
    console.log(`Porta: ${db.port}`)
    console.log(`Database: ${db.dbName}`)
    console.log(`Conexão: ${db.connectionString}`)
    ```
  </Tab>
  <Tab title="MySQL">
    ```typescript
    const db = await nivo.databases.create({
      name: 'mysql-producao',

    ```
  </Tab>

## Listar e buscar

```typescript
// Listar todos os bancos do projeto
const databases = await nivo.databases.list()

// Buscar por ID
const db = await nivo.databases.get('db_id')

// Status atual
console.log(db.status) // 'running' | 'stopped' | 'starting' | 'error'
```

## Ciclo de vida

```typescript
// Parar (para cobrança de compute enquanto parado)
await nivo.databases.stop('db_id')

// Iniciar
await nivo.databases.start('db_id')

// Reiniciar
await nivo.databases.restart('db_id')
```

## Executar queries

Para rodar SQL, use a **connection string** do banco com o driver nativo da sua linguagem (`pg`, `mysql2`, `mongoose`, `ioredis`). O SDK não embute um ORM nem um executor de queries.

```typescript
import { Pool } from 'pg'

// Pegue a connection string pelo SDK
const connectionString = await nivo.databases.getConnectionString('db_id')

// Use o driver nativo
const pool = new Pool({ connectionString })

const { rows } = await pool.query(
  'SELECT * FROM usuarios WHERE ativo = $1 LIMIT 10',
  [true]
)
console.log(rows)
```

<Tip>
  A connection string também vem no retorno de `databases.create()` no campo `connectionString`. Guarde-a como secret na sua aplicação.
</Tip>

## Rede privada

Todo banco fica acessível de duas formas dentro de um projeto:

<CardGroup cols={2}>
  <Card title="Conexão pública" icon="globe">
    Endereço externo (host + porta mapeada). Use para acessar de fora do projeto — sua máquina, ferramentas de migração, BI.
  </Card>
  <Card title="Conexão privada" icon="lock">
    Endereço interno do projeto. Apps no **mesmo projeto** alcançam o banco pelo nome, sem passar pela internet — mais rápido e sem expor a porta.
  </Card>
</CardGroup>

O endpoint de conexão devolve os dois:

```typescript
const conn = await nivo.databases.getConnection('db_id')

console.log(conn.connectionString)          // público
console.log(conn.internalConnectionString)  // privado (apps do projeto)
console.log(`${conn.internalHost}:${conn.internalPort}`)
```

<Note>
  Use a `internalConnectionString` como `DATABASE_URL` dos apps que rodam no
  mesmo projeto. Ela só resolve dentro da rede privada do projeto.
</Note>

## Métricas e estatísticas

```typescript
const stats = await nivo.databases.stats('db_id')

console.log(`CPU: ${stats.cpuUsagePct}% de ${stats.cpuLimitVcpu} vCPU`)
console.log(`RAM: ${stats.memoryUsageMB} / ${stats.memoryLimitMB} MB`)
console.log(`Tamanho: ${stats.sizePretty}`)
console.log(`Conexões: ${stats.activeConnections}/${stats.totalConnections}`)
```

## Logs

```typescript
const { log, status } = await nivo.databases.logs('db_id', { tail: 200 })
console.log(log)
```

<Note>
  Backups automáticos diários estão disponíveis nos planos Pro e Builder e são gerenciados pelo dashboard.
</Note>

---

## Deletar

```typescript
await nivo.databases.delete('db_id')
```

<Warning>
  A deleção é permanente e imediata. Todos os dados são perdidos. Faça um backup pelo dashboard antes de deletar.
</Warning>

## Referência completa

| Método | Descrição |
|--------|-----------|
| `databases.list()` | Lista bancos do projeto. |
| `databases.get(id)` | Busca banco por ID. |
| `databases.create(opts)` | Provisiona novo banco. |
| `databases.delete(id)` | Remove banco permanentemente. |
| `databases.start(id)` | Inicia o banco. |
| `databases.stop(id)` | Para o banco. |
| `databases.restart(id)` | Reinicia o banco. |
| `databases.getConnectionString(id)` | Retorna a connection string pública. |
| `databases.getConnection(id)` | Retorna conexão pública + privada (rede do projeto). |
| `databases.stats(id)` | Métricas de CPU, RAM, tamanho e conexões. |
| `databases.logs(id, opts)` | Retorna logs do banco. |
