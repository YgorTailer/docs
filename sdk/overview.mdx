## Dois clientes, dois contextos

O `@nivo-lat/sdk` expõe dois clientes com responsabilidades distintas:

| Cliente | Chave | Contexto | Recursos |
|---------|-------|----------|----------|
| `createClient()` | Publishable `nivo_pub_...` | Frontend, browser, app mobile | Auth de usuários finais, sessões, e-mail e SMS |
| `new NivoAPI()` | Secret `nivo_secret_...` | Backend, scripts, workers, CI/CD | Apps, deploys, bancos de dados, storage e todas as operações de infraestrutura |

<Warning>
Nunca exponha uma Secret Key no browser, app mobile ou em qualquer código acessível pelo cliente final.
</Warning>

## Operações de infraestrutura

Use `NivoAPI` no servidor para gerenciar toda a infraestrutura da sua aplicação.

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!)

// Criar e fazer deploy de uma aplicação
const app = await nivo.apps.create({
  name: 'minha-api',
  runtime: 'node20',
  startCmd: 'node dist/server.js',
  buildCmd: 'npm run build',
  installCmd: 'npm ci',
  ramMb: 512,
})

await nivo.apps.commitAndWait(app.id, './build.zip', 'build.zip', {
  onLog: (chunk) => process.stdout.write(chunk),
})

console.log(`App disponível em: https://${app.subdomain}.nivo.lat`)
```

## Deploy via GitHub

```typescript
const app = await nivo.apps.create({
  name: 'minha-api',
  sourceType: 'github',
  repoFullName: 'org/minha-api',
  repoBranch: 'main',
  runtime: 'node20',
  startCmd: 'node dist/server.js',
})

await nivo.apps.deployAndWait(app.id, {
  onLog: (chunk) => process.stdout.write(chunk),
})
```

## Auth de usuários finais

Use `createClient()` no frontend para autenticar usuários da sua aplicação.

```typescript
import { createClient } from '@nivo-lat/sdk'

const client = createClient('https://api.nivo.lat', 'nivo_pub_...')

// Login com e-mail e senha
const { session } = await client.auth.signIn.withPassword({
  email: 'usuario@exemplo.com',
  password: 'senha123',
})
```

## Recursos disponíveis

| Recurso | Operações |
|---------|-----------|
| **Apps** | `create`, `update`, `delete`, `start`, `stop`, `restart`, `deploy`, `commit`, `stats`, `logs` |
| **Env vars** | `getEnv`, `updateEnv`, `replaceEnv`, `removeEnv` |
| **Arquivos** | `listFiles`, `readFile`, `writeFile`, `deleteFile` |
| **Bancos de dados** | `create`, `query`, `start`, `stop`, `restart`, `stats`, `logs`, `tables` |
| **Storage** | Buckets, upload, upload multipart, signed URLs, CDN |
| **Auth** | Sign up, sign in, magic link, OAuth, sessões, usuários |

## Tratamento de erros

Todos os erros de API retornam uma instância de `NivoError` com status HTTP, mensagem e detalhe.

```typescript
import { NivoAPI, NivoError } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!)

try {
  const app = await nivo.apps.get('app-id')
} catch (err) {
  if (err instanceof NivoError) {
    console.error(`[${err.status}] ${err.message}`)
    if (err.detail) console.error(err.detail)
  }
}
```
