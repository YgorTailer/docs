# Nivo SDK Overview

The official SDK is published as [`@nivo-lat/sdk`](https://www.npmjs.com/package/@nivo-lat/sdk).

## Base URL

All SDK endpoints live under your API base URL (default `https://api.nivo.lat/api`).

## API versioning

| Prefix | Status |
|--------|--------|
| `/api/sdk/v1/*` | **Current** — use this for new integrations |
| `/api/sdk/*` (no version) | **Deprecated** — still works, unchanged behavior |

The `NivoAPI` client defaults to `apiVersion: 'v1'`, which calls `/api/sdk/v1/*`.

For legacy integrations that must keep using unversioned routes:

```ts
const nivo = new NivoAPI(apiKey, { projectId, apiVersion: null })
```

Direct `NivoClient` usage without `apiVersion` continues to target `/api/sdk/*` (legacy).

## Authentication

SDK routes require a **secret** project API key (`x-api-key` header) and project scope (`x-project-id` header). The client sets both automatically.

## Rate limits

SDK routes (`/api/sdk` and `/api/sdk/v1`) have a dedicated limit of **1000 requests/minute** per API key (fallback: client IP). This is separate from the global API limit (500/minute).
