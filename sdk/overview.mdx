## Um pacote, dois clientes

O pacote `@nivo-lat/sdk` nao e duplicado. Ele expoe dois clientes para casos diferentes:

| Cliente | Chave | Onde usar | Recursos |
|---------|-------|-----------|----------|
| `createClient()` | Publishable key `nivo_pub_...` | Browser/app do usuario final | Auth, sessoes, mail e SMS seguros para frontend. |
| `new NivoAPI()` | Secret key `nivo_secret_...` | Backend, scripts, managers e CI/CD | Apps, deploys, bancos, storage, CDN e operacoes administrativas. |

Para um bot manager, use `NivoAPI` no backend do manager. Nunca exponha secret keys no browser.

## Exemplo: manager criando app para um cliente

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!)

const app = await nivo.apps.create({
  name: 'cliente-vendas',
  sourceType: 'zip',
  runtime: 'node20',
  startCmd: 'npm start',
  buildCmd: 'npm run build',
  installCmd: 'npm ci',
  ramMb: 256,
  envVars: {
    CUSTOMER_ID: 'cus_123'
  }
})

await nivo.apps.commitAndWait(app.id, './app.zip', 'app.zip', {
  onLog: (chunk) => process.stdout.write(chunk)
})
```

## Fluxo GitHub

```typescript
const app = await nivo.apps.create({
  name: 'cliente-vendas',
  sourceType: 'github',
  repoFullName: 'empresa/cliente-vendas',
  repoBranch: 'main',
  runtime: 'node20',
  startCmd: 'npm start'
})

await nivo.apps.deployAndWait(app.id)
```

## Recursos principais

| Recurso | Exemplos |
|---------|----------|
| Apps | `create`, `commit`, `deploy`, `start`, `stop`, `restart`, `stats`, `logs`, `updateRam`, `delete`. |
| Env vars | `getEnv`, `updateEnv`, `replaceEnv`, `removeEnv`. |
| Bancos | `create`, `query`, `start`, `stop`, `restart`, `stats`, `logs`, `updateRam`, `delete`. |
| Storage | Buckets, upload, signed URLs, CDN, multipart upload e status do storage. |

## Erros

Respostas HTTP com erro viram `NivoError`.

```typescript
import { NivoAPI, NivoError } from '@nivo-lat/sdk'

try {
  await nivo.apps.get('app_invalido')
} catch (err) {
  if (err instanceof NivoError) {
    console.log(err.status)
    console.log(err.message)
    console.log(err.detail)
  }
}
```
