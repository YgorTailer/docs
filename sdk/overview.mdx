  </Tab>

O SDK é publicado como [`@nivo-lat/sdk`](https://www.npmjs.com/package/@nivo-lat/sdk) com suporte a **ESM** e **CJS** e tipagem TypeScript completa.

---

## Início rápido

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  projectId: process.env.NIVO_PROJECT_ID!
})

// Apps
const apps = await nivo.apps.list()

// Bancos de dados
const databases = await nivo.databases.list()

// Storage
const buckets = await nivo.storage.buckets.list()
```

## Autenticação

O SDK usa uma **API key secreta de projeto** enviada via header `x-api-key`. O `projectId` é enviado via header `x-project-id`. O cliente configura ambos automaticamente.

<Tabs>
  <Tab title="Variável de ambiente (recomendado)">
    ```typescript
    const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
      projectId: process.env.NIVO_PROJECT_ID!
    })
    ```

    No seu `.env`:
    ```bash .env
    NIVO_API_KEY=nivo_sk_prod_xxxxxxxxxxxxxxxxxxxx
    NIVO_PROJECT_ID=proj_abc123
    ```
  </Tab>
  <Tab title="Direto no código (não recomendado)">
    ```typescript
    const nivo = new NivoAPI('nivo_sk_prod_xxxx', {
      projectId: 'proj_abc123'
    })
    ```

    <Warning>
      Nunca exponha a API key no código client-side ou em repositórios públicos.
    </Warning>
  </Tab>
</Tabs>

## Versionamento da API

| Prefixo | Status |
|---------|--------|
| `/api/sdk/v1/*` | **Atual** — use para novas integrações |
| `/api/sdk/*` (sem versão) | **Depreciado** — ainda funciona, sem mudanças |

O cliente `NivoAPI` usa `v1` por padrão. Para usar a versão legada:

```typescript
const nivo = new NivoAPI(apiKey, {
  projectId,
  apiVersion: null  // usa /api/sdk/* sem versão
})
```

## Rate limits

| Rota | Limite |
|------|--------|
| `/api/sdk/v1/*` | 1.000 req/min por API key |
| Fallback (sem API key) | 500 req/min por IP |
