## Construtor

```typescript
new NivoAPI(apiKey: string, options: NivoClientOptions)
```

### Opções

<ParamField body="apiKey" type="string" required>
  API key secreta do projeto. Obtida no dashboard em **Configurações → API Keys**.
</ParamField>

<ParamField body="options.projectId" type="string" required>
  ID do projeto. Encontrado na URL do dashboard ou nas configurações do projeto.
</ParamField>

<ParamField body="options.baseUrl" type="string" default="https://api.nivo.lat/api">
  URL base da API. Altere apenas em ambientes de desenvolvimento local.
</ParamField>

<ParamField body="options.apiVersion" type="string | null" default="v1">
  Versão da API. Use `null` para a rota legada sem versão (`/api/sdk/*`).
</ParamField>

---

## Exemplos

<Tabs>
  <Tab title="Produção">
    ```typescript
    import { NivoAPI } from '@nivo-lat/sdk'

    const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
      projectId: process.env.NIVO_PROJECT_ID!
    })
    ```
  </Tab>
  <Tab title="Desenvolvimento local">
    ```typescript
    const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
      projectId: process.env.NIVO_PROJECT_ID!,
      baseUrl: 'http://localhost:3001/api'
    })
    ```
  </Tab>
  <Tab title="API legada">
    ```typescript
    const nivo = new NivoAPI(apiKey, {
      projectId,
      apiVersion: null  // usa /api/sdk/* sem versão
    })
    ```
  </Tab>
</Tabs>

## Tratamento de erros

O SDK lança exceções com detalhes do erro da API:

```typescript
try {
  const app = await nivo.apps.get('app_inexistente')
} catch (error) {
  if (error.status === 404) {
    console.log('App não encontrado')
  } else if (error.status === 401) {
    console.log('API key inválida')
  } else {
    throw error
  }
}
```

---

<Note>
  Use sempre variáveis de ambiente para a API key. Nunca exponha a chave no código client-side (browser) ou em repositórios públicos.
</Note>
