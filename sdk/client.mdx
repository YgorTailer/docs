---
title: "Configurando o Cliente"
description: "Escolha entre o cliente publico de Auth e o cliente administrativo de infraestrutura."
---

## Cliente administrativo

Use `NivoAPI` somente em backend, scripts locais, workers e CI/CD.

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  baseUrl: 'https://api.nivo.lat/api'
})
```

`NIVO_API_KEY` deve ser uma Secret Key (`nivo_secret_...`).

## Cliente publico

Use `createClient()` no browser para Auth e sessoes do usuario final.

```typescript
import { createClient } from '@nivo-lat/sdk'

const client = createClient('https://api.nivo.lat', 'nivo_pub_...')

await client.auth.signIn.withPassword({
  email: 'user@example.com',
  password: 'secret'
})
```

## Regra de seguranca

| Chave | Pode ir para frontend? | Uso |
|-------|-------------------------|-----|
| `nivo_pub_...` | Sim | Auth e recursos publicos. |
| `nivo_secret_...` | Nao | Managers, deploys, apps, bancos e storage administrativo. |

Nunca envie uma Secret Key para o browser, app mobile ou cliente final.
