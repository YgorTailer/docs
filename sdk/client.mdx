## NivoAPI

Use `NivoAPI` exclusivamente no servidor: backends, workers, scripts e pipelines de CI/CD.

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  projectId: process.env.NIVO_PROJECT_ID!
})
```

Para apontar para uma API customizada:

```typescript
const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  baseUrl: 'https://api.nivo.lat/api',
  projectId: process.env.NIVO_PROJECT_ID!
})
```

## API key

A API key é criada em **Minha conta → Configurações → API Keys** e usada junto com `projectId`.

| Item | Valor |
|------|-------|
| Prefixo | `nivo_key_...` |
| Escopo | Conta do usuário |
| Projeto | Informado no SDK por `projectId` |
| Ambiente | Backend, workers, scripts e CI/CD |

<Warning>
Mantenha a API key em variáveis de ambiente do servidor.
</Warning>

## Variáveis de ambiente recomendadas

```bash
# .env do servidor
NIVO_API_KEY=nivo_key_...
NIVO_PROJECT_ID=seu-project-id
```

## Acesso a arquivos privados

Arquivos em bucket privado são lidos pelo backend:

```typescript
const buffer = await nivo.storage.downloadFile('file-id')
```

Depois do download, responda o conteúdo pela rota da sua aplicação.
