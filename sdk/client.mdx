## NivoAPI

Use `NivoAPI` exclusivamente no servidor: backends, workers, scripts e pipelines de CI/CD.

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  projectId: process.env.NIVO_PROJECT_ID!
})
```

Para apontar para uma API customizada:

```typescript
const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  baseUrl: 'https://api.nivo.lat/api',
  projectId: process.env.NIVO_PROJECT_ID!
})
```

## API key

A Nivo usa uma única API key da conta, criada em **Minha conta → Configurações → API Keys**.

| Item | Valor |
|------|-------|
| Prefixo | `nivo_key_...` |
| Escopo | Conta do usuário |
| Projeto | Informado no SDK por `projectId` |
| Uso seguro | Backend, workers, scripts e CI/CD |
| Uso proibido | Browser, app mobile ou código acessível ao usuário final |

<Warning>
Não existe mais chave publicável e chave secreta separadas. Trate toda API key da Nivo como segredo de servidor.
</Warning>

## Variáveis de ambiente recomendadas

```bash
# .env do servidor
NIVO_API_KEY=nivo_key_...
NIVO_PROJECT_ID=seu-project-id
```

## Acesso a arquivos privados

Arquivos em bucket privado não recebem URL pública. Use o SDK no backend:

```typescript
const buffer = await nivo.storage.downloadFile('file-id')
```

Se sua aplicação precisar mostrar o arquivo para um usuário final, seu backend deve autenticar esse usuário, chamar o SDK e responder o conteúdo.
