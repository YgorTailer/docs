## NivoAPI — cliente de infraestrutura

Use `NivoAPI` exclusivamente no servidor: backends, workers, scripts e pipelines de CI/CD.

```typescript
import { NivoAPI } from '@nivo-lat/sdk'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!)
```

Para apontar para uma URL customizada:

```typescript
const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  baseUrl: 'https://api.nivo.lat/api',
})
```

A variável `NIVO_API_KEY` deve conter uma **Secret Key** (`nivo_secret_...`), obtida no painel da Nivo em **Configurações → API Keys**.

## createClient — cliente de Auth

Use `createClient()` no frontend para autenticar e gerenciar sessões dos usuários da sua aplicação.

```typescript
import { createClient } from '@nivo-lat/sdk'

const client = createClient('https://api.nivo.lat', process.env.NEXT_PUBLIC_NIVO_KEY!)
```

A variável deve conter uma **Publishable Key** (`nivo_pub_...`). Ela pode ser exposta no browser com segurança.

## Referência de chaves

| Tipo | Prefixo | Pode ir para o frontend? | Uso |
|------|---------|--------------------------|-----|
| Publishable Key | `nivo_pub_...` | Sim | Auth e sessões de usuários finais |
| Secret Key | `nivo_secret_...` | **Não** | Toda operação de infraestrutura via `NivoAPI` |

<Warning>
Nunca inclua uma Secret Key em código que rode no browser, app mobile ou qualquer ambiente acessível pelo usuário final. Use variáveis de ambiente exclusivas do servidor.
</Warning>

## Variáveis de ambiente recomendadas

```bash
# .env (servidor — nunca no frontend)
NIVO_API_KEY=nivo_secret_...

# .env.local (frontend — Next.js, Vite, etc.)
NEXT_PUBLIC_NIVO_KEY=nivo_pub_...
```
