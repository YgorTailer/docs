## Listar apps

```typescript
const apps = await nivo.apps.list()
// App[]
```

## Buscar app

```typescript
const app = await nivo.apps.get('app_id')
```

## Criar app

```typescript
const app = await nivo.apps.create({
  name: 'minha-api',
  type: 'site',          // 'site' | 'worker'
  runtime: 'node20',
  startCmd: 'npm start',
  buildCmd: 'npm run build',
  installCmd: 'npm install',
  ramMb: 512
})
```

### Tipos de app

<Tabs>
  <Tab title="Site (HTTP)">
    Recebe tráfego HTTP/HTTPS. Recebe uma URL pública automaticamente.

    ```typescript
    const app = await nivo.apps.create({
      name: 'minha-api',
      type: 'site',
      runtime: 'node20',
      port: 3000,
      startCmd: 'node dist/index.js'
    })
    console.log(app.url) // https://minha-api.nivo.lat
    ```
  </Tab>
  <Tab title="Worker (background)">
    Roda em background sem porta HTTP. Ideal para filas, cron jobs e processamento.

    ```typescript
    const worker = await nivo.apps.create({
      name: 'worker-emails',
      type: 'worker',
      runtime: 'node20',
      startCmd: 'node worker.js'
    })
    ```
  </Tab>
</Tabs>

## Variáveis de ambiente

```typescript
// Define/atualiza variáveis (faz merge com as existentes)
await nivo.apps.updateEnv('app_id', {
  DATABASE_URL: 'postgres://...',
  REDIS_URL: 'redis://...',
  SECRET_KEY: 'super-secreto'
})

// Substitui TODAS as variáveis pelo conjunto informado
await nivo.apps.replaceEnv('app_id', { NODE_ENV: 'production' })

// Remove variáveis por chave
await nivo.apps.removeEnv('app_id', ['SECRET_KEY'])
```

<Note>
  Por segurança, o SDK **não** expõe leitura das variáveis de ambiente — secrets nunca trafegam de volta pela API. Gerencie os valores na sua própria fonte de verdade.
</Note>

<Warning>
  Após alterar variáveis de ambiente, um novo deploy é necessário para que as mudanças entrem em vigor.
</Warning>

## Rede privada (app ↔ app)

Apps do mesmo projeto se enxergam por uma rede privada e podem se chamar **pelo nome**, sem passar pela internet. Cada app expõe um endereço interno:

```typescript
const app = await nivo.apps.get('app_id')

console.log(app.internalHost) // ex.: minha-api
console.log(app.internalUrl)  // ex.: http://minha-api:3000
```

Para um app chamar outro, basta usar o `internalUrl` do destino:

```typescript
// Dentro do "app-web", chamando o "minha-api" do mesmo projeto
const res = await fetch('http://minha-api:3000/usuarios')
const data = await res.json()
```

<CardGroup cols={2}>
  <Card title="Sites" icon="globe">
    O endereço interno é o subdomínio do app. Continuam recebendo tráfego público normalmente.
  </Card>
  <Card title="Workers" icon="gear">
    Não têm URL pública — só são alcançáveis pela rede privada do projeto, pelo nome.
  </Card>
</CardGroup>

<Note>
  O endereço interno só resolve entre apps e bancos do **mesmo projeto**. Para
  expor um app ao público, use um domínio (site) — workers permanecem privados.
</Note>

## Ciclo de vida

```typescript
// Parar
await nivo.apps.stop('app_id')

// Iniciar
await nivo.apps.start('app_id')

// Reiniciar
await nivo.apps.restart('app_id')
```

## Atualizar configuração

```typescript
await nivo.apps.update('app_id', {
  ramMb: 1024,
  startCmd: 'node dist/server.js'
})
```

---

## Domínio personalizado

```typescript
// Adicionar domínio (retorna instruções de DNS)
const result = await nivo.apps.domains.add('app_id', 'meusite.com.br')
console.log(result.verification)   // { type, name, value }

// Listar domínios
const domains = await nivo.apps.domains.list('app_id')

// Remover domínio (por id do domínio)
await nivo.apps.domains.remove('app_id', 'domain_id')
```

---

## Logs e métricas

```typescript
// Logs de runtime
const logs = await nivo.apps.logs('app_id', { tail: 200 })

// Métricas do container
const stats = await nivo.apps.stats('app_id')
console.log(`CPU: ${stats.cpuPct}%`)
console.log(`RAM: ${stats.memUsageMb} / ${stats.memLimitMb} MB`)
console.log(`Uptime: ${stats.uptimeSeconds}s`)
```

---

## Deletar app

```typescript
await nivo.apps.delete('app_id')
```

<Warning>
  A deleção é permanente. O app, seus logs e deployments são removidos imediatamente.
</Warning>

## Referência completa

| Método | Descrição |
|--------|-----------|
| `apps.list()` | Lista todos os apps do projeto. |
| `apps.get(id)` | Busca um app por ID. |
| `apps.create(opts)` | Cria um novo app. |
| `apps.update(id, opts)` | Atualiza configurações do app. |
| `apps.delete(id)` | Remove o app permanentemente. |
| `apps.start(id)` | Inicia o app. |
| `apps.stop(id)` | Para o app. |
| `apps.restart(id)` | Reinicia o app. |
| `apps.deploy(id)` | Dispara um novo deploy. |
| `apps.upload(id, file)` | Faz deploy enviando um ZIP. |
| `apps.updateEnv(id, vars)` | Define/atualiza variáveis (merge). |
| `apps.replaceEnv(id, vars)` | Substitui todas as variáveis. |
| `apps.removeEnv(id, keys)` | Remove variáveis por chave. |
| `apps.logs(id, opts)` | Retorna logs do container. |
| `apps.stats(id)` | Retorna métricas de CPU, RAM e uptime. |
| `apps.deployments(id)` | Lista deployments do app. |
| `apps.domains.add(id, domain)` | Adiciona domínio personalizado. |
| `apps.domains.list(id)` | Lista domínios do app. |
| `apps.domains.remove(id, domainId)` | Remove domínio personalizado. |
