## Deploy por ZIP

Envie o código diretamente pelo SDK sem usar o CLI. O `upload` recebe um ZIP (Buffer, caminho ou stream) e retorna o `deploymentId`.

```typescript
import { readFileSync } from 'fs'

// Gere o ZIP do projeto primeiro (archiver, jszip, etc.)
const zip = readFileSync('./dist/meu-app.zip')

const { deploymentId } = await nivo.apps.upload('app_id', zip, 'app.zip')
console.log(`Deployment: ${deploymentId}`)
```

## Deploy + espera com logs ao vivo

O `uploadAndWait` envia o ZIP e acompanha o build até terminar, com callbacks de log e status.

```typescript
const result = await nivo.apps.uploadAndWait('app_id', zip, 'app.zip', {
  onLog: (chunk) => process.stdout.write(chunk),
  onStatus: (status) => console.log(`→ ${status}`),
})

console.log(`Status final: ${result.status}`)  // success | error | cancelled
```

## Redeploy (GitHub / imagem Docker)

Para apps com `sourceType: "github"` ou `"image"`, dispare um novo deploy sem reenviar arquivos:

```typescript
const { deploymentId } = await nivo.apps.deploy('app_id')
await nivo.apps.waitForDeployment('app_id', deploymentId, {
  onLog: (chunk) => process.stdout.write(chunk),
})
```

## Listar deployments

```typescript
const { data } = await nivo.apps.deployments('app_id', { limit: 20 })

for (const dep of data) {
  console.log(`${dep.id} — ${dep.status} — ${dep.createdAt}`)
}
```

## Logs de um deployment

```typescript
const logs = await nivo.apps.deploymentLogs('app_id', 'deployment_id')
console.log(logs.log)
```

## Cancelar, repetir e reverter

```typescript
await nivo.apps.cancelDeployment('app_id', 'deployment_id')
await nivo.apps.retryDeployment('app_id', 'deployment_id')
await nivo.apps.rollbackDeployment('app_id', 'deployment_id')
```

## Caso de uso: CI/CD personalizado

```typescript
import { NivoAPI } from '@nivo-lat/sdk'
import { readFileSync } from 'fs'
import { createZip } from './utils/zip'

const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
  projectId: process.env.NIVO_PROJECT_ID!
})

async function deploy() {
  console.log('Criando ZIP...')
  const zipPath = await createZip('./dist')

  console.log('Fazendo deploy...')
  const result = await nivo.apps.uploadAndWait(
    process.env.NIVO_APP_ID!,
    readFileSync(zipPath),
    'app.zip',
    { onLog: (chunk) => process.stdout.write(chunk) },
  )

  if (result.status !== 'success') {
    throw new Error(`Deploy falhou com status: ${result.status}`)
  }
  console.log('Deploy concluído com sucesso!')
}

deploy().catch((err) => {
  console.error(err)
  process.exit(1)
})
```

## Referência

| Método | Descrição |
|--------|-----------|
| `apps.upload(id, file, filename?)` | Envia ZIP e inicia o deploy. |
| `apps.uploadAndWait(id, file, filename?, opts?)` | Envia ZIP e aguarda o build. |
| `apps.deploy(id)` | Dispara redeploy (GitHub/imagem). |
| `apps.deployAndWait(id, opts?)` | Redeploy e aguarda o build. |
| `apps.waitForDeployment(id, deploymentId, opts?)` | Acompanha um deploy em andamento. |
| `apps.deployments(id, opts?)` | Lista deployments. |
| `apps.deploymentLogs(id, deploymentId)` | Logs de um deployment. |
| `apps.cancelDeployment(id, deploymentId)` | Cancela um deploy. |
| `apps.retryDeployment(id, deploymentId)` | Repete um deploy. |
| `apps.rollbackDeployment(id, deploymentId)` | Reverte para um deploy anterior. |
