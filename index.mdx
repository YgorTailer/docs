## Como funciona

<Steps>
  <Step title="Crie um projeto no dashboard">
    Acesse [nivo.lat](https://nivo.lat), crie seu workspace e abra um projeto. Copie o `projectId` na URL ou nas configurações.
  </Step>
  <Step title="Instale o CLI ou SDK">
    <CodeGroup>
      ```bash CLI
      npm install -g @nivo-lat/cli
      ```
      ```bash SDK
      npm install @nivo-lat/sdk
      ```
    </CodeGroup>
  </Step>
  <Step title="Autentique">
    <CodeGroup>
      ```bash CLI
      nivo login
      # Abre o navegador — confirme o código exibido no terminal
      ```
      ```typescript SDK
      import { NivoAPI } from '@nivo-lat/sdk'

      const nivo = new NivoAPI(process.env.NIVO_API_KEY!, {
        projectId: process.env.NIVO_PROJECT_ID!
      })
      ```
    </CodeGroup>
  </Step>
  <Step title="Deploy ou crie recursos">
    <CodeGroup>
      ```bash CLI — App
      nivo link        # cria o app e salva o appId no .nivo
      nivo deploy . --watch
      ```
      ```typescript SDK — Banco de dados
      const db = await nivo.databases.create({
        name: 'producao',
        engine: 'postgres',
        version: '16'
      })
      console.log(`${db.host}:${db.port}`)
      ```
      ```typescript SDK — Storage
      const bucket = await nivo.storage.createBucket({
        name: 'uploads',
        public: true
      })
      const { file } = await nivo.storage.upload(bucket.id, './logo.png')
      console.log(file.url)
      ```
    </CodeGroup>
  </Step>
</Steps>

## Erros da API

<AccordionGroup>
  <Accordion title="2xx — Sucesso">
    | Status | Significado |
    |--------|-------------|
    | `200` | OK — requisição concluída |
    | `201` | Criado — recurso criado com sucesso |
    | `204` | Sem conteúdo — operação sem body de resposta |
  </Accordion>
  <Accordion title="4xx — Erros do cliente">
    | Status | Significado |
    |--------|-------------|
    | `400` | Requisição inválida — parâmetros incorretos |
    | `401` | Não autorizado — API key ausente ou inválida |
    | `403` | Proibido — plano não permite esta ação |
    | `404` | Não encontrado — recurso não existe |
    | `409` | Conflito — recurso já existe |
    | `429` | Rate limit atingido — aguarde `Retry-After` |
  </Accordion>
  <Accordion title="5xx — Erros do servidor">
    | Status | Significado |
    |--------|-------------|
    | `500` | Erro interno do servidor |
    | `502` | Bad gateway — node temporariamente indisponível |
    | `503` | Serviço indisponível — tente novamente |
  </Accordion>
</AccordionGroup>

## Rate limits

| Tipo | Limite |
|------|--------|
| Leitura `GET` | 1.000 req/min |
| Escrita `POST / PATCH / DELETE` | 300 req/min |
| Queries SQL diretas | 60 req/min |
| Upload de arquivos | 100 MB/arquivo |
| SDK `/api/sdk/v1` | 1.000 req/min por API key |

<Note>
  Quando o limite é atingido, a API retorna `429 Too Many Requests` com o header `Retry-After` indicando quantos segundos aguardar.
</Note>
