O Nivo é uma plataforma de infraestrutura para times que precisam hospedar aplicações, bancos de dados e arquivos de forma simples, sem gerenciar servidores.

## O que você pode fazer

<CardGroup cols={3}>
  <Card title="Hospedar Apps" icon="rocket">
    Deploy de aplicações Node.js, Python e outras runtimes com um comando.
  </Card>
  <Card title="Bancos de Dados" icon="database">
    PostgreSQL, MySQL, MongoDB e Redis prontos em segundos.
  </Card>
  <Card title="Storage & CDN" icon="cloud">
    Buckets de arquivos com URLs públicas e CDN global.
  </Card>
</CardGroup>

## Erros da API

Todos os erros retornam JSON com o campo `error` e o HTTP status correspondente.

| Status | Significado |
|--------|-------------|
| `200` | OK — requisição concluída |
| `201` | Criado — recurso criado com sucesso |
| `400` | Requisição inválida — parâmetros incorretos |
| `401` | Não autorizado — API key ausente ou inválida |
| `403` | Proibido — plano não permite esta ação |
| `404` | Não encontrado — recurso não existe |
| `409` | Conflito — recurso já existe |
| `500` | Erro interno do servidor |

## Rate limits

| Limite | Endpoints |
|--------|-----------|
| `1.000 req/min` | Leitura (GET) |
| `300 req/min` | Escrita (POST, PATCH, DELETE) |
| `60 req/min` | Queries SQL diretas |
| `100 MB/arquivo` | Upload de arquivos |

<Note>
  Quando o limite é atingido, a API retorna `429 Too Many Requests` com o header `Retry-After`.
</Note>
